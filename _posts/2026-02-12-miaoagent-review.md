## 项目背景

大二上学期，在《智能信息处理》课程中接触到了多模态智能体的概念。课后组队参加了一个校内的 AI 黑客松比赛，主题是"情感陪伴智能体"。

我们的项目命名为 **MIAOAGENT**，核心思路是：

- 通过**摄像头**实时捕捉用户的面部表情
- 用**语音识别**分析用户的语气和语调
- 结合**文本对话**内容综合判断用户情绪
- 生成个性化、有温度的对话回复

## 技术栈

```
前端:  HTML + CSS + JS
后端:  Python Flask
AI:    PyTorch (表情识别) + Whisper (语音识别) + Ollama (对话生成)
```

## 踩坑记录

### 1. 表情识别模型太小，准确率不高

最初用了轻量模型 MobileNetV2，在真实场景下准确率只有 60% 左右。

**解决方案**：改用 ResNet18 并在 FER2013 数据集上从头训练，准确率提升到 75%。牺牲了一点推理速度，但体验好了很多。

### 2. 实时视频流处理卡顿

原始方案是每帧都做推理，CPU 直接跑满。

**解决方案**：改为每 0.5 秒采样一帧，并在 Flask 里用多线程处理。

### 3. 多模态融合

表情说"开心"但文字说"我好烦"——到底信哪个？

**解决方案**：给文字更高权重（0.6），表情辅助（0.4），因为文字的语义信息更丰富。

## 最终效果

虽然没拿到名次，但通过这个项目我真正理解了多模态系统的工程挑战——**模型效果和实时性的平衡、不同模态信息的融合、端到端系统的集成调试**，这些是课堂上学不到的。
