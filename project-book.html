图书推荐系统

基于协同过滤算法的个性化图书推荐引擎

Python Flask Scikit-learn NumPy Pandas SQLite Bootstrap
项目概述

本项目是大二《Python 程序设计》课程的期末大作业。基于 协同过滤（Collaborative Filtering） 算法实现了一个个性化图书推荐系统。 用户可以注册并对已读图书进行评分，系统会根据评分数据自动推荐相似用户喜欢的书籍，以及和用户已评高分书籍相似的图书。

算法设计

系统采用混合推荐策略，结合了两种协同过滤方法，取长补短：

# 混合推荐策略
def hybrid_recommend(user_id, n=10):
    # 1. UserCF：基于相似用户的评分预测
    user_pred = user_based_cf(user_id, top_n=50)

    # 2. ItemCF：基于相似物品的评分预测
    item_pred = item_based_cf(user_id, top_n=50)

    # 3. 加权融合（权重通过验证集网格搜索优化）
    final_score = 0.45 * user_pred + 0.55 * item_pred

    # 4. 去重 + 按预测分数排序取 Top-N
    return sorted(final_score.items(), key=lambda x: x[1], reverse=True)[:n]
Web 应用功能
  1. 用户注册/登录：使用 Flask-Login 管理用户会话，密码经过 bcrypt 哈希加密存储。
  2. 图书浏览与搜索：支持按书名、作者、分类进行模糊搜索，每本图书展示封面、简介和平均评分。
  3. 评分系统：用户可对图书进行 1~5 星评分，评分后实时更新图书均分和用户画像。
  4. 个性化推荐：首页展示 "为你推荐" 个性化列表，并标注推荐理由（"因为你看过《XXX》"或"和你有相似口味的读者也喜欢"）。
  5. 热门榜单：基于全校用户评分数据生成的实时热门图书排行榜。
数据来源与处理

图书元数据来自网络公开数据集 Goodreads Books Dataset，包含 5000+ 本中文和英文图书的标题、作者、分类、简介信息。 评分数据通过模拟生成（300 个虚拟用户 + 随机评分矩阵），以模拟真实系统的推荐场景。 在数据预处理阶段，过滤了评分次数少于 3 次的"冷门用户"和少于 5 次评分的"冷门图书"，有效降低了评分矩阵的稀疏度。

收获与总结

通过本项目，我掌握了推荐系统的核心算法原理——协同过滤、矩阵分解（SVD）以及冷启动问题的应对策略。 也体会到在实际工程中，数据稀疏性是推荐系统的最大挑战之一——稀疏的用户-物品评分矩阵会严重影响相似度计算的可靠性。

后续优化方向：引入基于内容的推荐（Content-Based）作为冷启动补充，尝试使用 矩阵分解（Matrix Factorization）替代传统相似度计算，并升级至 深度协同过滤（Neural Collaborative Filtering）。
