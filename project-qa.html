校园智能问答助手

基于 RAG 和大语言模型的云南大学校园场景智能问答系统

RAG LangChain Python ChromaDB FastAPI Ollama
项目概述

本项目是大三《智能信息处理》课程的课程项目。灵感来源于日常校园生活中，许多同学经常在新生群、课程群里反复询问相同问题——"图书馆几点关门？""这门课怎么选？""教学楼在哪？"。我们构建了一个基于 检索增强生成（RAG） 技术的智能校园问答助手，能够基于云南大学校园的真实信息，准确回答各类校园相关问题。

技术方案

系统采用检索增强生成（RAG）架构，先将校园相关文档（学生手册、课程目录、校园地图信息等）分块后通过 ChromaDB 向量数据库存储，用户提问时先检索最相关的文档片段，再结合大语言模型生成自然流畅的回答。 

# 系统架构
用户提问 → FastAPI 接口
    │
    ├─ 向量检索（ChromaDB）
    │   └─ 召回 top-5 最相关文档片段
    │
    ├─ 提示词组装（LangChain Prompt Template）
    │   └─ 注入校园知识上下文 + 系统指令
    │
    └─ 大模型生成回答（Ollama / qwen2.5:7b）
        └─ 流式返回答案到前端聊天界面

关键技术点
  1. 文档分块策略：使用 LangChain 的 RecursiveCharacterTextSplitter，按语义段落切分文档，块大小 512 tokens，重叠 50 tokens，确保每个分块保留完整语义。
  2. 向量化模型：采用 BAAI/bge-large-zh-v1.5 中文向量模型进行文本嵌入，在 C-MTEB 中文基准测试上表现优异。
  3. 本地化部署：使用 Ollama 在本地运行 qwen2.5:7b 大模型，无需联网，确保校园知识数据安全；系统支持 CPU 运行，适用于学生笔记本环境。
  4. 流式输出：前端聊天界面支持 SSE 流式响应，实现打字机效果的回答展示，提升用户体验。
收获与总结

通过本项目，我深入理解了 RAG 架构的核心思想——"先检索、再生成"可以有效缓解大模型的知识幻觉问题，让回答更加可靠和可追溯。此外，LangChain 提供的模块化工具链大大降低了构建复杂 AI 应用的开发门槛。

后续计划：扩大知识库覆盖范围至全校所有学院和行政部门，引入多轮对话记忆功能，让问答更加连贯自然；同时探索使用知识图谱增强实体关系的推理能力。
